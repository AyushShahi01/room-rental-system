# Render Deployment Guide

## Quick Setup Steps

### 1. Generate a Django Secret Key

Run this locally:

```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

Copy the output - you'll need it for Render.

### 2. Push to GitHub

Make sure all changes are committed:

```bash
git add .
git commit -m "Prepare for Render deployment"
git push origin main
```

### 3. Create PostgreSQL Database on Render

1. Go to [render.com](https://render.com)
2. Click **New +** → **PostgreSQL**
3. Fill in:
   - **Name:** room-rental-db
   - **Database:** room_rental
   - **User:** (auto-generated)
   - Leave other settings as default
4. Click **Create Database**
5. Copy the **External Database URL** (looks like: `postgresql://user:pass@host:5432/dbname`)

### 4. Create Web Service on Render

1. Click **New +** → **Web Service**
2. **Connect Repository:**
   - Select your GitHub repository
   - Choose `main` branch
   - Select "Yes" for "Render app.yaml" to use render.yaml

3. **Configure:**
   - **Name:** room-rental-api
   - **Environment:** Python
   - **Build Command:** `cd backend/project && pip install -r requirements.txt && python manage.py collectstatic --noinput --settings=project.settings.prod`
   - **Start Command:** `cd backend/project && gunicorn project.wsgi:application --bind 0.0.0.0:$PORT`
   - **Plan:** Free (for testing)

### 5. Add Environment Variables

In the Web Service settings, go to **Environment** and add:

| Key                      | Value                                    | Notes                    |
| ------------------------ | ---------------------------------------- | ------------------------ |
| `DJANGO_SETTINGS_MODULE` | `project.settings.prod`                  | Fixed                    |
| `DJANGO_SECRET_KEY`      | `<paste-generated-key>`                  | From step 1              |
| `DEBUG`                  | `False`                                  | Fixed                    |
| `DATABASE_URL`           | `<paste-database-url>`                   | From step 3              |
| `ALLOWED_HOSTS`          | `your-service-name.onrender.com`         | Auto-generated by Render |
| `CORS_ALLOWED_ORIGINS`   | `https://your-frontend-url.onrender.com` | Your frontend URL        |

### 6. Run Migrations After Deployment

After Render deploys (you'll see "Live" status), run:

```bash
# Get your API URL from Render dashboard (something like: https://room-rental-api.onrender.com)

# Run migrations
curl -X POST https://your-api-url.onrender.com/api/migrate/ \
  -H "Authorization: Bearer YOUR_TOKEN"

# Or via Django shell (if you set up a management command)
```

Alternative: Connect to Render PostgreSQL via SSH and run:

```bash
python manage.py migrate --settings=project.settings.prod
python manage.py createsuperuser --settings=project.settings.prod
```

## Testing Locally Before Deployment

```bash
# Test production settings locally
DJANGO_SETTINGS_MODULE=project.settings.prod \
DJANGO_SECRET_KEY=test-key \
DEBUG=False \
ALLOWED_HOSTS=localhost,127.0.0.1 \
DATABASE_URL=sqlite:///db.sqlite3 \
python manage.py runserver
```

## Troubleshooting

### "ModuleNotFoundError" during build

- Check that build command path is correct: `cd backend/project && ...`

### Database connection errors

- Verify DATABASE_URL is correct
- Check PostgreSQL database is created on Render
- Ensure psycopg2-binary is in requirements.txt

### Static files not loading

- Ensure `collectstatic` runs successfully in build command
- Check STATIC_ROOT and STATIC_URL in settings

### CORS errors on frontend

- Update CORS_ALLOWED_ORIGINS with your frontend URL
- Use full HTTPS URL, not just the domain

## Production Checklist

- [ ] DATABASE_URL environment variable set
- [ ] DJANGO_SECRET_KEY is a strong random value
- [ ] DEBUG = False
- [ ] ALLOWED_HOSTS includes your Render URL
- [ ] CORS_ALLOWED_ORIGINS configured for your frontend
- [ ] Migrations run successfully
- [ ] Static files collected
- [ ] Email backend configured (if needed)
